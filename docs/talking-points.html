1

基本操作

10分
進行形式
参加者は操作するが、提案は全て Decline する（後続チャプターへの干渉防止）。Accept すると Ch.3-4 で app.py の状態が変わり、脆弱性検出等に影響するため。
「操作感を掴むことが目的です。コード変更は Ch.2 以降で行います」と案内してください。

キーメッセージ

  • Gemini Code Assist は Gemini 2.5 Pro ベース、最大 100 万トークンのコンテキストウィンドウ
  • 3つのインタラクション: Tab 補完（最速）、インラインチャット（直接編集）、チャットパネル（議論・質問）
  • @ でファイルやフォルダを明示的にコンテキストに含められる

Copilot ユーザー向け補足

  • インラインの補完体験は似ている
  • コンテキストウィンドウのサイズと Agent モードが差別化ポイント
2

Context Engineering

15分

キーメッセージ

  • GEMINI.md はプロジェクトに追加できる最もレバレッジの高いファイル
  • 10分で書けて、チーム全員の AI 出力品質が改善される
  • AI が「実際に読む」オンボーディング文書と考える

階層構造の説明

~/.gemini/GEMINI.md会社全体の規約
プロジェクトルート GEMINI.mdプロジェクト固有のルール
サブディレクトリ GEMINI.mdモジュール固有の追加ルール

Before/After のポイント

  • 同じプロンプトでも出力が劇的に変わることを強調
  • 「新しいチャットセッションを開始してから」After を試すよう促す

styleguide.md の説明

  • GitHub 連携で PR が作成されると自動レビュー
  • 自然言語でルールを書くだけ
  • Gemini Code Review の仕組みを図で説明
3

Agentic Coding

18分

キーメッセージ

  • 「コードを書いてもらう」から「目標を伝えて計画をレビューする」へのパラダイムシフト
  • 計画のレビューステップがエンジニアの価値を発揮する場所
  • Agent モード + GEMINI.md = 一貫した出力品質

注意事項

  • 「フィーチャーブランチで使うこと」を推奨
  • 生成 AI の出力は毎回異なる → 参加者の画面が講師と違っても正常
  • 計画の内容が妥当かレビューする習慣をつけることが重要
  • DB スキーマ変更: プロンプトで CREATE TABLE IF NOT EXISTS を指示済み。既存データに影響しない
Agent の出力が期待と違った場合
話法例
「これは生成 AI なので、出力は毎回変わります。大事なのはワークフローを体験すること」
completed/ の参照コードを見せてフォローしてください。
4

セキュア開発

13分

キーメッセージ

  • AI はセキュリティの専門家を代替しないが、最初のパスのコストを劇的に下げる
  • セキュリティ知識をコード（styleguide.md）として定着させることが真の価値
  • 「見つけて直す」だけでなく「ルール化して自動チェック」まで行うのがポイント

脆弱性の解説

#脆弱性BeforeAfter
1 SQL Injection（search_tickets） f-string で SQL 組み立て パラメータ化クエリへ
2 SQL Injection（get_tickets フィルタ） status を直接埋め込み パラメータ化 + enum 検証
3 ステータス無検証 任意の文字列を受け付ける 許可リストで検証
4 debug=True Werkzeug デバッガー露出 環境変数で制御
5 ハードコード秘密鍵 ソースに直書き 環境変数から読み込み

補足紹介のポイント

  • Gemini CLI sandbox (gVisor): With/Without の比較図で説明
  • Model Armor: 入出力チェックでプロンプトインジェクションや機密データ漏洩を防止
  • いずれもハンズオンでは触らない、紹介のみ
W

Wrap-up

2分
3つの持ち帰りアクション
  1. 明日、自分のプロジェクトに GEMINI.md を追加する
  2. フィーチャーブランチで Agent モードを試す
  3. styleguide.md にセキュリティルールを追加する

トラブルシューティング

問題対処
Gemini の応答が遅い 「少し待ちましょう。その間に completed/ のコードを見てください」
Before/After の差が小さい 事前キャプチャしたスクリーンショットを見せる
Agent の出力が予期しないもの 「正常です。ワークフローの体験が目的です」→ completed/ を参照
pip install が失敗 venv が有効化されているか確認。Python バージョンを確認
ポート 5000 が使用中 macOS Monterey 以降は AirPlay が使用。flask run -p 5001 を案内
.gemini/ が見えない 隠しフォルダ。ターミナルで code .gemini/styleguide.md を案内
Ch.4 で脆弱性が少ない GEMINI.md の効果で Ch.3 の Agent が一部修正済み。「これが Context Engineering の効果です」と説明
